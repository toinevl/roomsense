# Trust & Transparency FAQ — Implementation Plan

**Goal:** Plain-language FAQ page addressing privacy concerns. De-risks every other pitch by proactively answering "What data do you track?" and "How is it used?"

**Architecture:** New static page `frontend/src/pages/trust.ts`. No API calls. Simple HTML layout.

---

## Task 1: Create trust.ts with FAQ content

**Objective:** Static page with 6-7 Q&A pairs on privacy/data handling.

**Files:**
- Create: `frontend/src/pages/trust.ts`

**Code:**
```typescript
// frontend/src/pages/trust.ts
import { html } from 'lit-html'

export function trust() {
  return html`
    <div class="trust-page">
      <header class="trust-header">
        <h1>Trust & Transparency</h1>
        <p class="subtitle">What RoomSense does and doesn't track</p>
      </header>

      <div class="faq">
        <section class="faq-item">
          <h3>Do you track individuals?</h3>
          <p>No. RoomSense counts occupancy only — it measures how many people are in a room at a given time. We do not collect names, IDs, badges, or any personal information.</p>
        </section>

        <section class="faq-item">
          <h3>Do you have cameras or facial recognition?</h3>
          <p>No. Data comes from Terabee People Counting sensors, which use infrared time-of-flight technology. They detect movement and body shapes, not faces or identifying features.</p>
        </section>

        <section class="faq-item">
          <h3>What data is collected and stored?</h3>
          <p>Room occupancy (count), timestamp, room name, and building location. For demonstration purposes, we also store mock calendar reservation data (subject line, organizer name, attendee count) to show how room usage compares to actual bookings.</p>
        </section>

        <section class="faq-item">
          <h3>Who can see the data?</h3>
          <p>Everyone on this page can see anonymized building-level and room-level occupancy trends. Staff and facilities teams have access to the full historical dataset to support space planning. No individual or team data is ever exposed.</p>
        </section>

        <section class="faq-item">
          <h3>How long is data kept?</h3>
          <p>Demonstration data is kept for 30 days for demo purposes. In production, retention policies would be set by your institution's IT governance and data retention requirements.</p>
        </section>

        <section class="faq-item">
          <h3>Can I opt out?</h3>
          <p>Since RoomSense counts occupancy only and does not track individuals, there is nothing to opt out of. However, institutions deploying RoomSense can choose which rooms to monitor and set their own privacy policies.</p>
        </section>

        <section class="faq-item">
          <h3>How is the code reviewed?</h3>
          <p>RoomSense is open-source. The code, architecture, and data handling are publicly reviewable. <a href="#architecture">See our architecture page</a> for details on how data flows and what is/isn't visible.</p>
        </section>
      </div>

      <section class="trust-footer">
        <h2>Questions?</h2>
        <p>If you have privacy or security concerns, please reach out. We take data trust seriously.</p>
      </section>
    </div>
  `
}
```

**Verify:** `pnpm --filter frontend test -- trust` (syntax check only).
